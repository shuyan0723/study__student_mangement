## 一、功能概述
### 1.1 背景与目标
- **背景**：教师人工分析成绩需耗费大量时间，且分析维度有限（如仅能统计分数，无法深度挖掘知识点薄弱项、学生分层问题）；
- **核心目标**：通过调用Kimi大模型API，实现成绩数据的智能化分析，输出结构化、可落地的分析报告，辅助教师优化教学策略；
- **非核心目标**：控制API调用成本，保障数据隐私，支持分析结果的多端查看与导出。

### 1.2 适用场景
| 场景编号 | 场景描述                                  | 适用角色 | 频率预估 |
|----------|-------------------------------------------|----------|----------|
| S01      | 月考/期中/期末后，分析全班学科成绩整体情况 | 授课教师 | 每月1-2次 |
| S02      | 针对低分学生群体，分析共性薄弱知识点      | 授课教师 | 每周1次  |
| S03      | 对比本班与平行班成绩，定位教学差距        | 授课教师 | 每月1次  |
| S04      | 生成面向家长的成绩解读（简化版报告）| 授课教师 | 每学期2次 |

|---------|-------------------------------------------|--------|----------|-------------------------------------------|

## 四、非功能需求
### 4.1 性能需求
| 性能指标         | 要求值                                   | 测试场景                                  |
|------------------|------------------------------------------|-------------------------------------------|
| API调用响应时间  | 90%请求≤10s，99%请求≤30s，超时时间=30s    | 单次分析500条学生记录                     |
| 并发支持         | 支持≥50名教师同时发起分析请求             | 期末成绩分析高峰期                        |
| 页面加载速度     | 分析记录列表加载≤2s，报告页面加载≤3s      | 教师端首次进入功能模块                    |
| 重试机制         | API调用失败后自动重试1次，间隔3s          | 网络波动/API临时不可用                    |

### 4.2 数据安全与隐私
1. **API密钥安全**：
   - 服务端存储：采用AES-256加密，密钥与配置分离存储；
   - 传输安全：调用Kimi API时使用HTTPS，禁止明文传输密钥；
   - 权限控制：仅管理员可查看/修改密钥，操作需日志记录。
2. **成绩数据安全**：
   - 脱敏规则：学生姓名、学号、身份证号等敏感信息禁止传入Kimi API；
   - 传输安全：服务端与前端之间的成绩数据传输需加密；
   - 存储安全：分析报告仅存储在系统内，禁止Kimi平台留存数据（需确认Kimi API隐私政策）。
3. **操作日志**：
   - 记录内容：操作人ID/姓名、操作时间、操作类型、分析范围、API调用状态、消耗Token数；
   - 保留周期：90天，到期自动归档。

### 4.3 兼容性需求
| 兼容类型         | 支持范围                                  | 测试要点                                  |
|------------------|-------------------------------------------|-------------------------------------------|
| 浏览器兼容       | Chrome 80+、Edge 80+、Firefox 78+、Safari 14+ | 报告展示格式、导出功能、图表渲染          |
| 终端兼容         | PC端（优先）、平板端（适配）、移动端（只读） | 报告查看、按钮点击、筛选操作              |
| 导出文件兼容     | Word 2016+、WPS、PDF阅读器、Excel 2016+   | 格式错乱、内容缺失、图表显示异常          |

### 4.4 可用性需求
1. **操作流程**：发起分析的步骤≤3步（选择范围→选择维度→提交）；
2. **提示文案**：所有操作有明确反馈（如「调用次数已达上限」「数据脱敏中」），异常提示需给出解决方案；
3. **容错性**：用户误操作（如未选维度提交）时，前端即时校验并提示，不发起无效API调用；
4. **易用性**：提供「新手引导」（首次使用时展示功能流程）、「帮助文档」（点击查看使用说明）。

## 五、技术实现细节
### 5.1 Kimi API调用规范
#### 5.1.1 基础信息
| 配置项           | 示例值                                    | 备注                                      |
|------------------|-------------------------------------------|-------------------------------------------|
| API接口地址      | https://api.moonshot.cn/v1/chat/completions | 以Kimi官方最新文档为准                    |
| 请求方式         | POST                                      | -                                          |
| Content-Type     | application/json                          | -                                          |
| 认证方式         | Bearer Token（API密钥）| 密钥由管理员配置，服务端拼接请求头        |

#### 5.1.2 请求参数模板（JSON）
```json
{
  "model": "moonshot-v1-8k", // 模型版本，按需选择
  "messages": [
    {
      "role": "system",
      "content": "你是一名专业的教育数据分析专家，擅长分析学生成绩数据并给出教学建议。分析报告需结构化、语言简洁、建议可落地，禁止使用专业术语堆砌，符合中小学/高校教学场景。"
    },
    {
      "role": "user",
      "content": "请分析以下成绩数据：{desensitized_data}。分析要求：1. 重点分析{selected_dimensions}；2. {custom_instruction}；3. 输出格式：标题+基础数据概览+分维度分析+教学建议；4. 禁止泄露任何学生敏感信息。"
    }
  ],
  "temperature": 0.3, // 随机性，0-1，越低越精准
  "max_tokens": 2000, // 最大返回字符数
  "stream": false // 非流式返回，便于解析
}
