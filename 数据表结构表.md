# 学生成绩管理系统数据表结构表

## 1. 用户表 (users)

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `VARCHAR(36)` | `PRIMARY KEY` | 用户唯一标识（UUID） |
| `username` | `VARCHAR(50)` | `UNIQUE NOT NULL` | 用户名（登录账号） |
| `password` | `VARCHAR(100)` | `NOT NULL` | 密码（加密存储） |
| `email` | `VARCHAR(100)` | `UNIQUE NOT NULL` | 邮箱地址 |
| `phone` | `VARCHAR(20)` | | 电话号码 |
| `role` | `ENUM('student', 'teacher', 'admin')` | `NOT NULL` | 用户角色 |
| `created_at` | `DATETIME` | `NOT NULL DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `DATETIME` | `NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

## 2. 学生表 (students)

| `college` | `VARCHAR(100)` | `NOT NULL` | 学院 |
| `major` | `VARCHAR(100)` | `NOT NULL` | 专业 |
| `home_address` | `VARCHAR(255)` | | 家庭地址 |
| `created_at` | `DATETIME` | `NOT NULL DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `DATETIME` | `NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

## 数据字典

### 用户角色枚举
- `student`: 学生
- `teacher`: 教师
- `admin`: 管理员

### 学生状态枚举
- `enrolled`: 已选课
- `completed`: 已完成
- `failed`: 不及格

### 教师状态枚举
- `active`: 在职
- `inactive`: 离职

### 成绩等级枚举
- `A`: 优秀 (90-100)
- `B`: 良好 (80-89)
- `C`: 中等 (70-79)
- `D`: 及格 (60-69)
- `F`: 不及格 (0-59)

### 通知类型枚举
- `system`: 系统通知
- `academic`: 学术通知
- `event`: 活动通知

### 通知优先级枚举
- `low`: 低优先级
- `medium`: 中优先级
- `high`: 高优先级

### 日志状态枚举
- `success`: 操作成功
- `failed`: 操作失败
- `warning`: 操作警告

## 数据库设计说明

1. **主键设计**: 所有表使用UUID作为主键，确保数据的唯一性和分布式系统兼容性。

2. **外键关系**: 建立了完整的外键约束，确保数据的一致性和完整性。

3. **冗余字段**: 适当使用冗余字段（如teacher_name）提高查询性能。

4. **JSON字段**: 使用JSON字段存储课程ID列表等复杂数据结构。

5. **时间戳**: 所有表都包含created_at和updated_at字段，记录数据的生命周期。

6. **枚举类型**: 使用枚举类型确保数据的合法性和一致性。

7. **索引策略**: 对于频繁查询的字段（如username、student_id、course_id等），建议创建索引以提高查询性能。

8. **数据安全**: 密码字段使用加密存储，敏感信息需要适当的访问控制。
